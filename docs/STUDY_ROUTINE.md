# 자습 1시간 복습 루틴

수업 끝나고 학원 자습시간(1시간)에 **그날치를 끝내는** 루틴. 빠르게 복습 노트를 남기고,
그 노트가 그대로 문서·문제·발표 덱의 **저작 소스**가 되게 한다.

## ⏱️ 오늘 자습 60분 (v2)

| 시간 | 할 일 | 주체 |
|------|-------|------|
| **0~5분** | **워밍업**: 어제 문제 3개 — `python check_quiz.py days/<어제>/quiz.json --warmup`. 정답 오류 발견 시 quiz.json 즉시 수정(2분 상한) | 손 |
| **5~15분** | **개념 제목 잡기**: review.md에 뼈대 나열 + **"오늘의 축" 한 줄** | 손 |
| **15~32분** | **실습 명령어 정리**: 실행 순서대로 + 결과 요지 | 손 |
| **32~45분** | **에러·흐름·왜**. 문서 오류/보강 거리는 `[오류]`/`[보강]` **메모만, 고치지 않기** | 손 |
| **45~52분** | **5줄 요약** — 최후 사수선 | 손 |
| **52~58분** | **quiz.json 3~5문제**: "정답은 X, 근거는 이 문장"을 **내가 지정** → AI는 **오답 보기만** 생성 | AI 보조 |
| **58~60분** | **출제자 검수**: `python check_quiz.py days/<N>/quiz.json --doc <문서>` → 시트로 문제당 10초, 정답 인덱스 + 해설-원문 일치 확인 | 손 |

> 원칙: 손이 먼저(개념·명령어·요약은 내 손으로), AI는 보조(오답 생성·배선·검수 시트). 매뉴얼 대비 `[건너뛴 것]`은 32~45분 메모에 함께.
> 출제자 검수(58~60분)는 **허브에 들어간(들어갈) 문제를 검수**하는 것 — 새로 만드는 게 아니다. quiz.json이 허브의 원본이므로 check_quiz가 그 역할.

### 일일 복습 시트 (days/N/compare.md)

수업 노트가 오면 Claude가 **하루를 한 장으로 접은 복습 시트**를 만든다 —
오늘의 축 · 개념 제목 · 실습 명령어(순서+결과 요지) · 5줄 요약 · 건너뛴 것.

**읽는 곳** (md 원본 대신 보기 좋게 내장):
- **허브 '복습' 탭** — 일차 칩으로 골라 보기 (`inject_compare.py`, make_day가 자동)
- **자료실 하단 '04 일일 복습'** — 일차별 일지 페이지 (`reviews/dayN.html`, build_library가 자동)

**복습 동선(10분)**: ①축 한 줄을 소리 내어 설명 → ②설명 안 되는 개념만 `[문서 §]` 링크로 이동해 재독
→ ③명령어 표의 결과 숫자를 가리고 떠올리기 → ④허브에서 그날 문제 풀기.

**연결 규약** — Claude가 시트를 쓸 때 각 개념 옆에 `[문서 §x.y](assets/문서.html#앵커)` 링크를 달고,
건너뛴 것에는 매뉴얼·"질문 후 기록" 링크를 단다. 막히는 항목은 클릭 한 번으로 그 자리에서 복습.

### 52~58분 — "정답은 내가, 오답은 AI" 규약

review.md의 `[문제 초안]`에 이렇게 적는다:

```
Q1. NL에서 바깥 테이블을 고르는 기준은?
    정답: 바깥 행 수를 최소로 (필터 센 쪽, 없으면 작은 쪽)
    근거: "driving = 바깥 행 수를 최소로 만드는 쪽" (§7.7)
```

→ 이 대화(Claude)에 주면: **오답 보기 3개 생성 + 정답 인덱스 세팅 + 근거 문장을 해설로** 가공해 quiz.json을 만든다. AI는 정답을 만들지 않는다 — **출제 의도와 근거는 항상 사람 소유.**

### 58~60분 — 출제자 검수 (check_quiz.py)

```bash
python check_quiz.py days/74/quiz.json --doc assets/sql_tuning.html
#  → ✓/✗ 구조(정답 인덱스 범위·보기 수·id 중복·일차 일치) — 틀리면 주입 차단
#  → out/day74.quizcheck.md 검수 시트: 문제마다 ✔정답 표시 → 10초씩 눈으로
#  → ⚠ 근거 대조: 해설의 숫자가 원문에 없으면 표시(계산값이면 OK로 넘김)
```

`make_day.py`가 주입 전에 자동 실행한다(구조 오류 시 중단). 시트에서 볼 것 딱 둘: **✔가 진짜 정답인가 · 해설이 원문과 맞는가.**

## 📝 매일 남길 양식

`days/N/review.md` 에 아래를 채운다 (복사용 원본: `days/TEMPLATE.md`).

```
[오늘 주제]      예: Archive Log / Data Pump / Undo / Recovery / Index
[핵심 개념]      오늘 배운 개념을 3줄로
[주요 명령어]    명령어 3~5개
[확인 뷰]        v$log, v$datafile, dba_tablespaces 같은 뷰
[실습 흐름]      1. 무엇을 확인했다  2. 무엇을 변경했다  3. 어떤 결과가 나왔다
[헷갈린 점]      오늘 이해 안 된 부분 1개
[건너뛴 것]      매뉴얼(Concepts 등) 이 주제 챕터에서 수업이 안 다룬 소주제 + 추정 이유
```

### 건너뛴 것 — 왜 남기나, 어떻게 다루나

수업은 매뉴얼의 부분집합이다. **무엇이 빠졌는지 모르면 "다 배웠다"고 착각**한다.
그날 주제의 매뉴얼 챕터 목차를 훑어(5분) 안 다룬 소주제를 적고, 이유를 **추정 분류**로 표시한다:

| 분류 | 뜻 | 예 |
|------|-----|----|
| ① 뒤 차시 | 커리큘럼 순서상 나중에 다룸 | 조인 수업에서 옵티마이저 통계는 다음 주 |
| ② 비중 낮음 | 시험·실무 비중이 낮아 생략 | 잘 안 쓰는 옵션·레거시 문법 |
| ③ 구버전·폐기 | 최신 버전에서 폐기·대체됨 | rule-based optimizer |
| ④ 내부 심화 | 운영에 불필요한 내부 구현 | 래치 내부 알고리즘 |
| ⑤ 환경 제약 | 실습 환경에서 재현 불가 | RAC·Exadata 전용 기능 |
| ? 모름 | 이유를 모르겠음 → **확인 필요** | 강사에게 질문 |

- **"왜"는 추정이다** — 강사가 이유를 말 안 했으면 지어내지 말고 분류+`?`로 남긴다.
- `?`와 ①은 **행동으로 이어진다**: `?`→강사 질문, ①→뒤 차시에서 회수 확인.
- 저작 시: 덱 **빈틈 & 궁금한 점** 슬라이드와 문서 **확인 권장 tip**의 재료가 된다(아래 지도).

## 🚀 빠른 시작 (자습 준비)

```bash
python new_day.py 73          # days/73/ 생성 (review.md + quiz.json 뼈대)
#   → days/73/review.md 를 위 양식대로 채운다 (1시간)
```

채운 뒤, 이 대화(Claude)에서 저작·배선:

```bash
# 1) 대화로 저작: review.md 를 근거로
#    - assets/<topic>.html 에 그날 섹션 추가 (새 사실 X, 원문만 구조화)
#    - days/73/quiz.json 채우기 (MCQ/ESSAY/TERMS/PLANQ)
#    - days/73/73.slides.json (일차 덱, 선택)
# 2) 배선:
python make_day.py 73 --doc <topic>.html
```

## 🔗 양식이 파이프라인으로 이어지는 지도

복습 노트의 각 칸은 산출물의 특정 부분이 된다 — **양식을 채우는 게 곧 저작 1단계**다.

| 양식 칸 | → 발표 덱 | → 문서(HTML) | → 문제 |
|---------|-----------|--------------|--------|
| `[오늘 주제]` | 덱 **축**(중심 주제) · 섹션 | 문서 섹션 제목 | 태그/일차 |
| `[핵심 개념]` | **큰 그림** 슬라이드 · 개념 슬라이드 | `<h3 class="blk">` 개념 | TERMS |
| `[주요 명령어]` | **code 블록** | `<pre class="term">` | PLANQ 재료 |
| `[확인 뷰]` | plan/뷰 슬라이드 | 표·`.tip` | MCQ 보기 |
| `[실습 흐름]` | **steps · 실측 플랜** | `.why`(왜) | PLANQ/ESSAY |
| `[헷갈린 점]` | **빈틈 & 질문** 슬라이드 | `.tip`(확인 권장) | MCQ/ESSAY |
| `[건너뛴 것]` | **빈틈 & 질문** 슬라이드(매뉴얼 대비 빈 곳 + 추정 이유) | `.tip`(확인 권장, 지어내지 않음) | — (배우면 그때 문제화) |

> 축 잡기: `[주요 명령어]`·`[실습 흐름]`에서 **가장 자주·중요하게 반복된 것**이 그날의 축이다(→ `docs/DECK_DESIGN.md`).

## 규칙
- **원문만** — 노트에 없는 새 사실은 문서·덱에 넣지 않는다(확인 필요는 '확인 권장'으로 표시).
- **매일 커밋** — `days/N/` 는 git 추적이라 세션이 바뀌어도 남는다.
- 노트는 개인용이라 **자료실(공개)엔 안 올린다**(덱만 공개).
