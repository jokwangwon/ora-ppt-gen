# 자습 1시간 복습 루틴

수업 끝나고 학원 자습시간(1시간)에 **그날치를 끝내는** 루틴. 빠르게 복습 노트를 남기고,
그 노트가 그대로 문서·문제·발표 덱의 **저작 소스**가 되게 한다.

## ⏱️ 1시간 타임박스

| 시간 | 할 일 | 남기는 곳 |
|------|-------|-----------|
| **0~10분** | 오늘 배운 **개념 제목** 정리 | `[오늘 주제]` · `[핵심 개념]` |
| **10~30분** | **실습 명령어** 다시 정리 (직접 쳐보며) | `[주요 명령어]` · `[확인 뷰]` |
| **30~45분** | **에러 / 흐름 / 왜 하는지** 정리 | `[실습 흐름]` · `[헷갈린 점]` |
| **45~60분** | 오늘 내용 **5줄 요약** (설명하듯) | `[핵심 개념]` 보강 + 말로 설명해보기 |

> 원칙: **손으로 다시 쳐보고**(10~30분), **왜 하는지**를 한 줄로(30~45분), **남에게 설명하듯**(45~60분).

## 📝 매일 남길 양식

`days/N/review.md` 에 아래를 채운다 (복사용 원본: `days/TEMPLATE.md`).

```
[오늘 주제]      예: Archive Log / Data Pump / Undo / Recovery / Index
[핵심 개념]      오늘 배운 개념을 3줄로
[주요 명령어]    명령어 3~5개
[확인 뷰]        v$log, v$datafile, dba_tablespaces 같은 뷰
[실습 흐름]      1. 무엇을 확인했다  2. 무엇을 변경했다  3. 어떤 결과가 나왔다
[헷갈린 점]      오늘 이해 안 된 부분 1개
```

## 🚀 빠른 시작 (자습 준비)

```bash
python new_day.py 73          # days/73/ 생성 (review.md + quiz.json 뼈대)
#   → days/73/review.md 를 위 양식대로 채운다 (1시간)
```

채운 뒤, 이 대화(Claude)에서 저작·배선:

```bash
# 1) 대화로 저작: review.md 를 근거로
#    - assets/<topic>.html 에 그날 섹션 추가 (새 사실 X, 원문만 구조화)
#    - days/73/quiz.json 채우기 (MCQ/ESSAY/TERMS/PLANQ)
#    - days/73/73.slides.json (일차 덱, 선택)
# 2) 배선:
python make_day.py 73 --doc <topic>.html
```

## 🔗 양식이 파이프라인으로 이어지는 지도

복습 노트의 각 칸은 산출물의 특정 부분이 된다 — **양식을 채우는 게 곧 저작 1단계**다.

| 양식 칸 | → 발표 덱 | → 문서(HTML) | → 문제 |
|---------|-----------|--------------|--------|
| `[오늘 주제]` | 덱 **축**(중심 주제) · 섹션 | 문서 섹션 제목 | 태그/일차 |
| `[핵심 개념]` | **큰 그림** 슬라이드 · 개념 슬라이드 | `<h3 class="blk">` 개념 | TERMS |
| `[주요 명령어]` | **code 블록** | `<pre class="term">` | PLANQ 재료 |
| `[확인 뷰]` | plan/뷰 슬라이드 | 표·`.tip` | MCQ 보기 |
| `[실습 흐름]` | **steps · 실측 플랜** | `.why`(왜) | PLANQ/ESSAY |
| `[헷갈린 점]` | **빈틈 & 질문** 슬라이드 | `.tip`(확인 권장) | MCQ/ESSAY |

> 축 잡기: `[주요 명령어]`·`[실습 흐름]`에서 **가장 자주·중요하게 반복된 것**이 그날의 축이다(→ `docs/DECK_DESIGN.md`).

## 규칙
- **원문만** — 노트에 없는 새 사실은 문서·덱에 넣지 않는다(확인 필요는 '확인 권장'으로 표시).
- **매일 커밋** — `days/N/` 는 git 추적이라 세션이 바뀌어도 남는다.
- 노트는 개인용이라 **자료실(공개)엔 안 올린다**(덱만 공개).
